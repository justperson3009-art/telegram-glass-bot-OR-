# 🔧 ПОЛНАЯ ИЗОЛЯЦИЯ БОТОВ НА СЕРВЕРЕ

## Проблема

Команда `pkill -f "python.*main.py"` в старом `deploy.sh` убивала **ВСЕХ** ботов на сервере!

## Решение

Теперь каждый бот изолирован:
- ✅ Отдельный PID файл (`glass-bot.pid`)
- ✅ Уникальный путь (`/opt/glass-bot/`)
- ✅ Systemd сервис (опционально)

---

## Вариант 1: Через deploy.sh (простой)

Скрипт теперь убивает **ТОЛЬКО** glass-bot:

```bash
bash deploy.sh
```

---

## Вариант 2: Через systemd (рекомендуется!)

### Установка сервиса:

```bash
sudo cp /opt/glass-bot/glass-bot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable glass-bot
sudo systemctl start glass-bot
```

### Управление ботом:

```bash
# Статус
sudo systemctl status glass-bot

# Перезапуск
sudo systemctl restart glass-bot

# Остановить
sudo systemctl stop glass-bot

# Логи
journalctl -u glass-bot -f
```

### Обновление после изменений:

```bash
cd /opt/glass-bot
git pull origin main
sudo systemctl restart glass-bot
```

**Преимущества systemd:**
- ✅ Автозапуск при перезагрузке сервера
- ✅ Автоперезапуск при падении
- ✅ Не влияет на другие боты
- ✅ Удобные логи через `journalctl`
